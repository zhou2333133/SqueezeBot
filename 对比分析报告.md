# 对比分析报告
## SqueezeBot × CryptoPainter 两周路线

**日期**: 2026-05-13
**范围**: 架构设计、风控理念、AI 定位、进化策略
**说明**: 只学习对比，不修改代码

---

## 一、路线对比总览

| 阶段 | CryptoPainter | SqueezeBot |
|---|---|---|
| 前期 | AI + 自进化策略，让 Agent 自主调参 | 已有完整 6 层架构（妖币→AI→规则学习器→Evolver→risk_guard→执行） |
| 中期转折 | 05-10: 自进化尝试下 300 万美元单，被风控拦住 | 已有 LOCKED_PARAMS + boundary_config 双重保护 |
| 后期结论 | AI 降级为"秘书"，交易核心用纯代码 | 当前架构已符合"AI 不直接决策交易"原则 |

---

## 二、逐项对比

### 2.1 硬风控：必须独立存在，AI 不能改

| 对比项 | CryptoPainter | SqueezeBot |
|---|---|---|
| 经历 | 自进化为盈利尝试下 300 万刀单，靠风控拦截 | 未发生（有保护未触发） |
| 做法 | 后置：出事后才意识到需要硬风控 | 前置：设计阶段就内置 |
| 锁定参数 | 未明确提及 | ✅ `LOCKED_PARAMS` + `boundary_config.json` 双重锁定 |
| 开仓金额 | AI 可建议但应硬锁 | ✅ `SCALP_POSITION_USDT = 100`，LOCKED，Evolver 改不了 |
| 最大持仓 | 未明确 | ✅ `SCALP_MAX_POSITIONS = 3`，LOCKED |
| 杠杆 | 未明确 | ✅ `max_leverage = 10`，boundary_config 限制 |
| 每日亏损 | 未明确 | ✅ `daily_max_loss_usdt = 200`，boundary_config + check_open_order |

**结论**: SqueezeBot 在硬风控上**领先于他的认知阶段**。他是在吃了亏才意识到需要硬风控，而 SqueezeBot 初始架构就内置了多层硬边界。

### 2.2 AI 定位：能用纯代码的不要交给 AI

| 对比项 | CryptoPainter | SqueezeBot |
|---|---|---|
| 最终结论 | AI 降级为秘书：监控、分析、报告、辅助 | ✅ 完全一致 |
| 行情采集 | 纯代码 | ✅ 纯代码（WS + REST，无 AI 参与） |
| 信号生成 | 传统算法产生基础信号 | ✅ `_detect_signal_v3` 纯规则引擎 |
| 市场状态 | 未明确，疑似 AI 判断 | ✅ `market_state.py` 纯规则判断（EMA/ATR/成交量等） |
| 规则选择 | 让 Agent 自主决策 | ✅ `rule_selector.py` 软判断，基于历史统计 |
| 参数调优 | AI 提 proposal，但出过 RSI 参数改反事故 | ✅ Evolver 提 proposal → risk_guard 审核 → 写入 |
| 风控决策 | 硬编码独立模块 | ✅ risk_guard 独立于 Evolver |
| 中文报告 | 未明确 | ✅ `ai_reporter.py` MiniMax 生成日报 |

**结论**: 高度一致。SqueezeBot 在"AI 不染指交易核心"上走得更远 — market_state 用纯规则而非 LLM，rule_selector 用统计而非 AI 判断。

### 2.3 自进化策略：触发条件、边界、审计

| 对比项 | CryptoPainter | SqueezeBot |
|---|---|---|
| 触发条件 | 连续止损/回撤/盈利超阈值时才进化 | ✅ Evolver 有 `EVOLVER_MIN_TOTAL_TRADES`（15 笔）+ `EVOLVER_RUN_AFTER_CLOSED_TRADES`（30 笔）门槛 |
| 样本门槛 | 未明确定量 | ✅ 代码级门槛，不够样本直接跳过 |
| 参数修改流程 | AI 提修改，无明确中间审核层 | ✅ Evolver proposal → risk_guard.apply_proposals() → clamp + 校验 → 写入 |
| 修改风险 | RSI 参数改反导致反向开单一晚 | ✅ PARAM_BOUNDS 限制每参数 [min, max]，防止越界 |
| 审计追踪 | 未明确 | ✅ `param_patches.jsonl` + `guard_events.jsonl` 双重审计 |
| 回滚 | 手动恢复 | ✅ `rollback_last_policy()` 函数 |

**结论**: SqueezeBot 的进化系统在安全防护上更完善 — 每层都有独立审计，proposal 必须过 risk_guard。

### 2.4 Memory 管理

| 对比项 | CryptoPainter | SqueezeBot |
|---|---|---|
| 问题 | Memory.md 容易膨胀，需要每天清理 | ✅ `learning_memory.json` 按 `candidate_source × symbol × state_key × rule_id` 聚合，不会无限膨胀 |
| 结构 | 未明确，疑似线性日志 | ✅ 结构化统计：`last_20` 滚动窗口防止老数据拖累 |
| 清理机制 | 手动 | ✅ `score` 评分机制自动降权低分规则，不删除数据 |

**结论**: SqueezeBot 的结构化记忆系统比他描述的 `Memory.md` 更可控。

### 2.5 CryptoPainter 有而 SqueezeBot 没有的

| 优点 | 说明 | 是否值得借鉴 |
|---|---|---|
| 多平台部署经验 | Hermes 比 OpenClaw 更稳定 | ✅ 但属运维，非架构 |
| RSI 参数保护 | 他的教训是"单参数越调越极端" | ✅ 我们的 PARAM_BOUNDS 已有 [min,max]，但可检查单参的极端化趋势 |
| 亏损单时间限制 | "亏损单 4 小时内自动止损" | ⚠️ 我们已有 SL + 时间止损，但无独立的"持仓亏损超时"硬逻辑 |

### 2.6 SqueezeBot 有而 CryptoPainter 没有的

| 优点 | 说明 |
|---|---|
| 6 层分层架构 | 妖币扫描→AI→规则学习器→Evolver→risk_guard→execution_router，每层各司其职 |
| LOCKED_PARAMS + PARAM_BOUNDS | 双重锁定，AI 连越界的可能性都没有 |
| 冷启动保护 | `rule_selector` 样本 < 10 不拦截，< 30 只 warn |
| 因果回溯 | `param_attribution.feed_trade()` 追踪每次参数修改后的 N 笔交易 |
| 候选来源维度 | `learning_memory` 按 candidate_source 聚合，知道哪些来源表现差 |
| state_key 体系 | `market_state` 输出统一 state_key，learning_memory 按状态聚合 |

## 三、关键差异总结

### 最一致的方面（方向高度重合）
1. **AI 不能直接下单** — 两人都认同
2. **硬风控必须独立** — 两人都认同
3. **策略核心用代码而非 AI** — 两人都认同
4. **AI 做分析/报告/监控** — 两人都认同
5. **自进化需要边界和审计** — 两人都认同

### 最大差异（SqueezeBot 超前之处）
1. **设计 vs 补救** — SqueezeBot 的 6 层架构在设计阶段就内置了风控隔离；他是吃了亏才补
2. **规则学习器** — 我们没有把规则选择交给 LLM，而是用 `learning_memory` 做冷启动保护的结构化统计
3. **state_key 体系** — `market_state.classify()` 输出的统一状态键让回测和聚合有确定基准

### SqueezeBot 的潜在薄弱点
1. **未经历实盘大资金检验** — CryptoPainter 在 05-10 遇到了放大资金后的回撤，我们还没遇到
2. **锁的边界可能太死** — `SCALP_POSITION_USDT = 100` 固定，如果行情波动大可能需要用户手动调整
3. **LLM 报告的质量依赖 MiniMax** — 如果 API 不稳定或模型变化，日报可能不可用

## 四、建议（按优先级）

| 优先级 | 建议 | 理由 |
|---|---|---|
| P3 | 增加「单参数极端化检测」 | CryptoPainter 遇到 RSI 越调越极端的问题，PARAM_BOUNDS 有 [min,max] 但可以加连续同向变化的告警 |
| P3 | 持仓亏损超时独立硬逻辑 | 现有 SL + 时间止损，但可增加一个独立的"盈利未达标超时平仓"选项 |

**注意**: 这不是我们当前的薄弱项，只是可选的防御性增强。CryptoPainter 的核心教训我们基本都已经覆盖了。

---

## 五、一句话结论

> CryptoPainter 这两周是从"AI 自主交易"到"AI 做秘书"的转型，而 SqueezeBot 在设计之初就已经是这个结论的工程实现。我们的差距不在架构，在实盘运行时间和资金规模的经验积累。
